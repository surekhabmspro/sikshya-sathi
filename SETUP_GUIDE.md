# शिक्षा साथी — पूर्ण रूपमा निःशुल्क डिप्लोयमेन्ट गाइड

यो गाइडले तपाईंको Teacher Teaching Assistant System लाई **पूर्णतया निःशुल्क** सेवाहरू प्रयोग गरी इन्टरनेटमा राख्ने (deploy गर्ने) तरिका देखाउँछ — कुनै पनि क्रेडिट कार्ड नचाहिने।

## प्रयोग हुने निःशुल्क सेवाहरू

| आवश्यकता | सेवा | निःशुल्क सीमा |
|---|---|---|
| डाटाबेस + Auth + Realtime sync | Supabase | 500MB डाटाबेस, असीमित API request |
| फाइल भण्डारण (PDF, PPT, भिडियो) | Supabase Storage | 1GB |
| वेबसाइट होस्टिङ (PWA) | Vercel वा Netlify | असीमित (व्यक्तिगत प्रयोगका लागि) |
| एआई शिक्षण सहायक | Claude API (pay-as-you-go, धेरै सस्तो) वा बिना AI | प्रयोग अनुसार भुक्तानी, तर एक शिक्षकको प्रयोगमा महिनाको १ डलर भन्दा कम |

यो स्ट्याक तपाईंको प्रयोग धेरै नबढेसम्म सधैं $0 मा चल्छ।

---

## चरण १: Supabase प्रोजेक्ट बनाउनुहोस्

1. https://supabase.com मा जानुहोस् र "Start your project" क्लिक गर्नुहोस्
2. GitHub वा इमेलले साइन अप गर्नुहोस् (निःशुल्क)
3. नयाँ प्रोजेक्ट बनाउनुहोस् — नाम राख्नुहोस् "sikshya-sathi"
4. एउटा बलियो डाटाबेस पासवर्ड सेट गर्नुहोस् र सुरक्षित ठाउँमा लेखेर राख्नुहोस्
5. क्षेत्र (Region) को रूपमा भारत वा सिंगापुरको नजिकको क्षेत्र छान्नुहोस् (नेपालका लागि छिटो हुन्छ)

## चरण २: डाटाबेस स्किमा चलाउनुहोस्

1. Supabase ड्यासबोर्डमा बायाँतिर "SQL Editor" क्लिक गर्नुहोस्
2. `supabase-schema.sql` फाइलको सबै सामग्री कपी गरी पेस्ट गर्नुहोस्
3. "Run" थिच्नुहोस् — यसले सबै तालिका (chapters, lessons, materials, questions, homework, journal, आदि) र सुरक्षा नियमहरू (RLS) बनाउँछ

## चरण ३: Authentication सक्षम गर्नुहोस्

1. "Authentication" > "Providers" मा जानुहोस्
2. "Email" provider सक्षम गर्नुहोस् (शिक्षकले इमेल र पासवर्डले लगइन गर्नेछन्)
3. वैकल्पिक: "Phone" provider पनि सक्षम गर्न सकिन्छ यदि OTP-आधारित लगइन चाहिएमा

## चरण ४: फाइल भण्डारण (Storage) बनाउनुहोस्

1. "Storage" मा जानुहोस्
2. नयाँ bucket बनाउनुहोस्, नाम: `materials`
3. यसलाई **Private** राख्नुहोस् (Public होइन) ताकि अरूले फाइल हेर्न नसकून्
4. Storage > Policies मा गई owner-only access policy थप्नुहोस् (auth.uid() ले match गरेको फाइल मात्र हेर्न मिल्ने)

## चरण ५: API Keys लिनुहोस्

1. "Project Settings" > "API" मा जानुहोस्
2. यी दुई कुरा कपी गर्नुहोस्:
   - **Project URL** (जस्तै: `https://xxxxx.supabase.co`)
   - **anon public key**
3. यी दुईलाई पछि वेब एपको environment variables मा राख्नुहोस्

## चरण ६: एप कोड तयार पार्नुहोस्

हाल बनेको प्रोटोटाइप (`teacher-assistant.jsx`) मा देखाइएको सबै स्क्रिनलाई वास्तविक डाटासँग जोड्नका लागि:

1. प्रोटोटाइपको in-memory डाटा (`LESSONS`, `MATERIALS`, `QUESTIONS` आदि array हरू) लाई Supabase बाट ल्याइने डाटाले बदल्ने
2. `@supabase/supabase-js` लाइब्रेरी प्रयोग गरी हरेक स्क्रिनमा डाटा read/write गर्ने
3. यो काम ठूलो भएकाले अर्को चरणको रूपमा गर्नु राम्रो हुन्छ — चाहनुभयो भने म यो पनि बनाइदिन्छु

## चरण ७: निःशुल्क होस्टिङमा राख्नुहोस्

**विकल्प A — Vercel (सिफारिस गरिएको):**
1. https://vercel.com मा GitHub खाताले साइन अप गर्नुहोस्
2. एपको कोड GitHub repository मा राख्नुहोस्
3. Vercel मा "Import Project" गरी त्यो repository छान्नुहोस्
4. Environment variables मा Supabase URL र key थप्नुहोस्
5. "Deploy" थिच्नुहोस् — केही सेकेन्डमै लाइभ हुन्छ, निःशुल्क `.vercel.app` डोमेन सहित

**विकल्प B — Netlify:** माथिकै प्रक्रिया, https://netlify.com मा

**विकल्प C — GitHub Pages:** स्थिर साइटका लागि उपयुक्त, तर Vercel/Netlify बढी सजिलो छ React एपका लागि

## चरण ८: मोबाइलमा PWA को रूपमा प्रयोग गर्नुहोस्

1. Vercel मा डिप्लोय भएको लिङ्क मोबाइलको Chrome/Safari मा खोल्नुहोस्
2. "Add to Home Screen" छान्नुहोस्
3. अब यो मोबाइलमा एउटा सामान्य एपजस्तै देखिन्छ र अफलाइनमा पनि काम गर्छ (PWA को सुविधाले)

## Android, iOS, Windows एपमा रूपान्तरण

PWA तयार भएपछि, यसलाई native एपमा बदल्न:
- **Android:** PWABuilder (https://www.pwabuilder.com) — निःशुल्क, Google Play मा पठाउन मिल्ने APK बनाउँछ
- **iOS:** PWABuilder ले Xcode प्रोजेक्ट पनि बनाउन सक्छ, तर App Store मा राख्न Apple Developer खाता ($99/वर्ष) चाहिन्छ
- **Windows:** PWABuilder ले Microsoft Store का लागि प्याकेज पनि बनाउँछ, धेरै हदसम्म निःशुल्क

## अर्को के गर्ने?

म तपाईंलाई यी मध्ये कुनै पनि बनाइदिन सक्छु:
1. प्रोटोटाइपलाई Supabase सँग वास्तविक रूपमा जोड्ने पूर्ण कोड (auth, CRUD, file upload)
2. फाइल अपलोड flow (शिक्षकले PDF/PPT तान्ने-छोड्ने/drag-drop)
3. AI सहायकलाई वास्तविक Claude API सँग जोड्ने (अपलोड गरिएको सामग्री मात्र पढ्ने गरी)
4. GitHub repository संरचना तयार पार्ने ताकि सिधै डिप्लोय गर्न मिलोस्
